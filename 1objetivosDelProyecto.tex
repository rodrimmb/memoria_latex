El objetivo de este proyecto es desarrollar una herramienta didáctica para facilitar a los alumnos que tengan que cursar asignaturas en las que se explique algoritmos de búsqueda, el funcionamiento del algoritmo de búsqueda en espacio de estados A estrella.

Para facilitar el acceso a esta herramienta desde cualquier dispositivo que posea un navegador web moderno, se decidió desarrollar esta herramienta en formato de página web. Según estudios de Cisco en 2012 había 8.700 millones de dispositivos con conexión a internet \citeotras{dispositivos_conectados}, esto hace que el alcance de la herramienta a sus potenciales usuarios sea enorme.

Uno de nuestro principales objetivos era que la herramienta fuera muy visual y fácil de entender, queríamos que cada paso fuera comprendido de un solo vistazo a través de gráficos sencillos y tablas.

\section{Objetivos Técnicos}
Técnicamente los objetivos del proyecto son dos: poner en práctica las técnicas de desarrollo más usadas actualmente por los mejores desarrolladores y aprender a usar las herramientas más modernas y populares del desarrollo web.

Respecto a las técnicas de desarrollo, el objetivo es tener un buen diseño software para poder enfrentarnos a la fase de mantenimiento de una manera más legible y sencilla así como conseguir crear nuevas funcionalidades sin tener que modificar en gran medida código antiguo. Los costes de mantenimiento pueden abarcar el 80\% de un proyecto de software por lo que hay que valorar un buen diseño.

Respecto a las herramientas de desarrollo teníamos varios objetivos. El principal era aprender un nuevo lenguaje de programación dinámico como es Ruby, que sea diferente a Java que es el principal lenguaje de programación usado en el máster y poder así complementar los conocimientos adquiridos durante la realización del mismo. 

También se pretende usar el lenguaje de scripting más popular del mundo, debido a que puede ser usado en los navegadores web, JavaScript. Este lenguaje ha sido usado en una de las asignaturas del máster y consideramos que sería bueno profundizar un poco más en su uso debido a su enorme popularidad.

Por último, y por seguir la línea innovadora a la hora de elegir las herramientas de desarrollo para el proyecto, nos decantamos por usar una base de datos que no usara el clásico esquema entidad relación de las bases de datos relacionales y decidimos elegir una base de datos NoSQL como es MongoDB, que no requiere un modelo estático y estructura de datos para almacenar la información.
